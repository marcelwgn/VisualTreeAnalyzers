Namespace VisualTreeAnalyzers.Accessibility.Rules

Classes

ControlNameNotIncludeTypeRule

This is an accessibility rule checking if the name of an element does not include the type of the datacontext.

ControlNonEmptyNameRule

This rule checks if the controls has a non empty name if it has a peer.

ExpectKeyboardFocusableFalseRule

This rule checks if the passed element is expected to not be keyboard focusable. See the ExpectKeyboardFocusableTrueRule for the opposite rule.

ExpectKeyboardFocusableTrueRule

This rule checks if the passed element is expected to be keyboard focusable. See the ExpectKeyboardFocusableFalseRule for the opposite rule.

LandmarkTypeMainOnceRule

This rule checks if only one element with the landmark type "Main" was encountered.

ListItemNameNotEqualDataContextRule

The purpose of this rule is to check whether the ListViewItems and GridViewItems have a valid name and are not using the type of the datacontext.

LocalizedLandMarkTypeRule

This rule checks whether the localized landmark type is specified if the element has a landmark type.

NameReasonableLengthRule

Determines whether the name of a control has a reasonable length, that is, it is not to short and not too long.

OrientationNotNoneRule

Verifies that the scrollable controls have an orientation.

PatternInterfaceImplementedRule

This rule verifies that every Windows.UI.Xaml.Automation.Peers.PatternInterface provided is actually implemented by the automation peer.

RequiredSelectionRule

This rule verifies that a container with required selection has an element selected. A failure for this rule might occur if the peer of the element does not properly list the selection.

SingleSelectionRule

Verifies that a container with single selection has at most one item selected. A failure for this rule might occur if the peer of the element does not properly list the selection.

TypeButtonIsInvokeOrToggleRule

This rule verifies that elements with automation type Button are either toggleable or invokable.
